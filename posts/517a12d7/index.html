Intro to Extended Euclidean Algorithm

S1l3ntC0nquer2024-09-302024-09-30

Prologue

I decided to write this to help myself to better understand the attacks in RSA or other crypto system. And if this can help you, that would be my honor! Also, all the code in this note will in Python since it’s the most used exploit script language in CTFs.

Let’s start!

Euclidean Algorithm

Intro

It’s an algorithm to calculate the GCD (Greatest Common Divisor) between 2 numbers, and in Gujarati, it’s called 輾轉相除法 BTW.

Principles

It’s an recursive algorithm, so every step’s output is the input of the next step. Here we set kkk is the count of the steps, starting from 0.

The inputs of every step are non-negative remainders rk−1,rk−2r_{k-1}, r_{k-2}rk−1​,rk−2​. And since the remainders obtained by each round are smaller than those calculated in the former round, rk−1<rk−2r_{k-1}<r_{k-2}rk−1​<rk−2​. In the kthk^{th}kth step, the algorithm calculates the quotient qkq_kqk​ and remainder rkr_krk​ that satisfy the following equations.

rk−2=qkrk−1+rkwhere 0≤rk≤rk−1r_{k-2} = q_{k} r_{k-1} + r_{k} \quad \text{where } 0 \leq r_{k} \leq r_{k-1}rk−2​=qk​rk−1​+rk​where 0≤rk​≤rk−1​

That is, rk−2r_{k-2}rk−2​ should recursively minus rk−1r_{k-1}rk−1​ until it’s less than rk−1r_{k-1}rk−1​.

In the first step (k=0k=0k=0), we set r−2r_{-2}r−2​ and r−1r_{-1}r−1​ are equal to aaa and bbb. In the second step (k=1k=1k=1), we calculate the quotient and the remainder by dividing r−1r_{-1}r−1​ (which is bbb) by r0r_0r0​ (the remainder we obtained in the first step), and so on. The whole algorithm can be represented by the following equations.

a=q0b+r0b=q1r0+r1r0=q2r1+r2r1=q3r2+r3   ⋮rn=0\begin{align*} a&= q_0 b+r_0 \\ b&=q_1 r_0+r_1 \\ r_0&=q_2 r_1+r_2\\ r_1&=q_3 r_2+r_3\\ &\,\,\, \vdots \\ r_n&=0 \end{align*}abr0​r1​rn​​=q0​b+r0​=q1​r0​+r1​=q2​r1​+r2​=q3​r2​+r3​⋮=0​

If a<ba<ba<b, the first step is actually switching those, since a÷b=q0=0…r0=aa\div{b}=q_0=0\dots r_0=aa÷b=q0​=0…r0​=a.

Because the remainder calculate in each step is always decreasing and never gonna be negative, there must be an nnn such that rn=0r_n=0rn​=0 in the nthn^{th}nth step. Then rn−1r_{n-1}rn−1​ is the GCD of aaa and bbb. Also, nnn is impossible to be infinite so there must be finite natural numbers between r0r_0r0​ and 000.

Code

def euclidean_algorithm(a, b) -> int:
    if (b == 0):
        return a
    return euclidean_algorithm(b, a%b)

This function will return the GCD of a,ba, ba,b.

Extended Euclidean Algorithm

Intro

Extended Euclidean Algorithm, obviously is the extension of Euclidean Algorithm. For known a,ba, ba,b, the Extended Euclidean Algorithm can find the x,yx, yx,y that satisfy the Bézout’s identity ax+by=gcd⁡(a,b)ax+by=\gcd(a, b)ax+by=gcd(a,b) while obtaining gcd⁡(a,b)\gcd(a, b)gcd(a,b). If a<0a<0a<0, we can change the problem into ∣a∣(−x)+by=gcd⁡(∣a∣,b)|a|(-x)+by=\gcd(|a|, b)∣a∣(−x)+by=gcd(∣a∣,b), and let x′=(−x)x'=(-x)x′=(−x).

Bézout’s identity is an important result in Number Theory, describing the relationship between the GCD of 2 numbers & their linear combinations. The equation is

ax+by=gcd⁡(a,b)ax+by=\gcd(a, b)ax+by=gcd(a,b)

For arbitrary 2 integers a,ba, ba,b, there must be integers x,yx, yx,y that satisfy this equation.

By the way, the Extended Euclidean Algorithm is an self-verifying algorithm. We can simply use the si+1s_{i+1}si+1​ and ti+1t_{i+1}ti+1​ obtained in the last step to times gcd⁡(a,b)\gcd(a, b)gcd(a,b), and see if they’re equal to aaa and bbb to verify the calculation is correct.

And the most important thing for us (CTFers, or cybersecurity enthusiasts) is that this algorithm can be used to calculate the modular multiplicative inverse, which is necessary in RSA to obtain the keys.

Principles

In an standard Euclidean Algorithm, we mark the 2 numbers for which we want to calculate the GCD as aaa and bbb, and the quotient we get in the ithi^{th}ith step to be qiq_iqi​, remainder to be ri+1r_{i+1}ri+1​. Then we can write the Eulidean Algorithm as following.

r0=ar1=b   ⋮ri+1=ri−1−qiriand0≤ri+1≤∣ri∣   ⋮\begin{align*} r_0&=a \\ r_1&=b \\ &\,\,\,\vdots \\ r_{i+1}&=r_{i-1}-q_i r_i\quad\text{and}\quad0\leq{r_{i+1}}\leq|r_i| \\ &\,\,\,\vdots \end{align*}r0​r1​ri+1​​=a=b⋮=ri−1​−qi​ri​and0≤ri+1​≤∣ri​∣⋮​

When some step the ri+1=0r_{i+1}=0ri+1​=0, the algorithm breaks. And the rir_iri​ obtained in the last step is gcd⁡(a,b)\gcd(a, b)gcd(a,b).

The Extended Euclidean Algorithm adds 2 additional sequences sis_isi​ and tit_iti​ based on qiq_iqi​ and rir_iri​, and lets s0=1s_0=1s0​=1, s1=0s_1=0s1​=0, t0=0t_0=0t0​=0 and t1=1t_1=1t1​=1. In the Extended Euclidean Algorithm, each step calculates ri+1=ri−1−qirir_{i+1}=r_{i-1}-q_i r_iri+1​=ri−1​−qi​ri​, and furthermore calculates si+1=si−1−qisis_{i+1}=s_{i-1}-q_i s_isi+1​=si−1​−qi​si​ and ti+1=ti−1−qitit_{i+1}=t_{i-1}-q_i t_iti+1​=ti−1​−qi​ti​, which is

r0=ar1=bs0=1s1=0t0=0t1=1   ⋮   ⋮ri+1=ri−1−qiriand 0≤ri+1<∣ri∣si+1=si−1−qisiti+1=ti−1−qiti   ⋮{\displaystyle {\begin{aligned}r_{0}&=a&r_{1}&=b\\s_{0}&=1&s_{1}&=0\\t_{0}&=0&t_{1}&=1\\&\,\,\,\vdots &&\,\,\,\vdots \\r_{i+1}&=r_{i-1}-q_{i}r_{i}&{\text{and }}0&\leq r_{i+1}<|r_{i}|\\s_{i+1}&=s_{i-1}-q_{i}s_{i}\\t_{i+1}&=t_{i-1}-q_{i}t_{i}\\&\,\,\,\vdots \end{aligned}}}r0​s0​t0​ri+1​si+1​ti+1​​=a=1=0⋮=ri−1​−qi​ri​=si−1​−qi​si​=ti−1​−qi​ti​⋮​r1​s1​t1​and 0​=b=0=1⋮≤ri+1​<∣ri​∣​

And the breaking condition is the same with the Euclidean Algorithm, which is ri+1=0r_{i+1}=0ri+1​=0, and the sis_isi​ and tit_iti​ at this time is the solution for gcd⁡(a,b)=ri=asi+bti\gcd(a, b)=r_i=as_i+bt_igcd(a,b)=ri​=asi​+bti​.

To find the modular multiplicative inverse a−1a^{-1}a−1, we can use the following steps.

  1. Confirm that aaa and bbb are coprime, which means gcd⁡(a,b)=1\gcd(a, b)=1gcd(a,b)=1, if it’s not, then the a−1a^{-1}a−1 doesn’t exist.
  2. Use the Extended Euclidean Algorithm to solve ax+by=1ax+by=1ax+by=1, then the xxx is what we want.

For example, if we want to find the modular multiplicative inverse of 3 modulo 11.

Extended Euclidean Part11=3⋅3+23=2⋅1+12=1⋅2+0Inverse Part1=3−1⋅21=3−1⋅(11−3⋅3)1=4⋅3−1⋅11So x=4 is the modular multiplicative inverse of 3.\begin{align*} &\text{Extended Euclidean Part} \nonumber \\\\ &11 = 3 \cdot 3 + 2 \nonumber \\ &3 = 2 \cdot 1 + 1 \nonumber \\ &2 = 1 \cdot 2 + 0 \nonumber \\\\ &\text{Inverse Part} \nonumber \\\\ &1 = 3 - 1 \cdot 2 \nonumber \\ &1 = 3 - 1 \cdot (11 - 3 \cdot 3) \nonumber \\ &1 = 4 \cdot 3 - 1 \cdot 11 \nonumber \\\\ &\text{So } x = 4 \text{ is the modular multiplicative inverse of } 3. \end{align*}​Extended Euclidean Part11=3⋅3+23=2⋅1+12=1⋅2+0Inverse Part1=3−1⋅21=3−1⋅(11−3⋅3)1=4⋅3−1⋅11So x=4 is the modular multiplicative inverse of 3.​

Code

def extended_euclidean(a: int, b: int) -> tuple[int, int, int]:
    if a == 0:
	    return (b, 0, 1)
    g, y, x = extended_euclidean(b % a, a)
    return (g, x - (b // a) * y, y)

def inverse(a: int, b: int) -> int:
	g, x, y = extended_euclidean(a, b) # ax + by = gcd
	if g == 1:
		return x % b
	raise ValueError("base is not invertible for the given modulus.")
