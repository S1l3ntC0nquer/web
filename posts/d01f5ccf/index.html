Web

Evil Calculator

Command Injection

First observe the problem, open the developer tools of F12, enter something in Random to see how it works. Here we first press 3+3 on the computer.

We can see in the picture that he actually passes a request to the back-end server. The back-end server will perform the calculation and send the result back to the front-end. The payload here looks like this:

{"expression": "3+3"}

We can find that he executed the subsequent 3+3. At this time, if we look at the app.py file given in the question, we can be more sure that our idea is correct. as follows:

@app.route("/calculate", methods=["POST"])
def calculate():
    data = request.json
    expression = data["expression"].replace(" ", "").replace("_", "")
    try:
        result = eval(expression) # 這裡就是我們要inject的地方！
    except Exception as e:
        result = str(e)
    return jsonify(result=str(result))

So we only need to replace "3+3" in {expression: "3+3"} with the command we want to inject and it can be executed.

I use the Chrome plug-in HackBar to send the request. I originally wanted to directly cat ../flag, but I found that its source code will replace spaces, like this

expression = data["expression"].replace(" ", "").replace("_", "")

So I changed the writing method and the payload is as follows:

{"expression": "''.join([open('../flag').read()])"}

Then we get flag!

babyRSA

This question is called babyRSA, but I still couldn’t write it out in the end 🥲🥲🥲. Okay, let’s take a look at the title first. The title is as follows:

import random
from Crypto.Util.number import getPrime

from secret import flag


def gcd(a, b):
    while b:
        a, b = b, a % b
    return a


def generate_keypair(keysize):
    p = getPrime(keysize)
    q = getPrime(keysize)
    n = p * q
    phi = (p - 1) * (q - 1)

    e = random.randrange(1, phi)
    g = gcd(e, phi)
    while g != 1:
        e = random.randrange(1, phi)
        g = gcd(e, phi)
    d = pow(e, -1, phi)
    return ((e, n), (d, n))


def encrypt(pk, plaintext):
    key, n = pk  # pk =(e, n)
    cipher = [pow(ord(char), key, n) for char in plaintext]
    return cipher


def decrypt(pk, ciphertext):
    key, n = pk  # pk = (d, n)
    plain = [chr(pow(char, key, n)) for char in ciphertext]
    return "".join(plain)


public, private = generate_keypair(512)
encrypted_msg = encrypt(public, flag)
decrypted_msg = decrypt(private, encrypted_msg)

print("Public Key:", public)
print("Encrypted:", encrypted_msg)
print("Decrypted:", decrypted_msg)

The question looks like a normal RSA encryption process:

  1. First obtain two large prime numbers ppp and qqq
  2. Multiply ppp and qqq to get nnn
  3. (p−1)×(q−1)(p-1)\times(q-1)(p−1)×(q−1) gets ϕ(n)\phi(n)ϕ(n)
  4. Find another eee that is coprime with ϕ(n)\phi(n)ϕ(n). At this point, the public key has been completed.
  5. Then use e−1 mod ϕ(n)e^{-1} \bmod \phi(n)e−1modϕ(n) to calculate d. At this point, the private key has been found.

Then I will sort out the various solutions I have tried this time.

Violent factorization

I tried to use FactorDB to decompose nnn and find the required ppp and qqq, but unfortunately it ended in failure.

Fermat’s factorization

Fermat decomposition is used when the difference between ppp and qqq is very small, but because I don’t know ppp and qqq, I’ll give it a try. As for the My script, it looks like this:

def fermat_factorization(n: int) -> tuple[int, int]:
    a = gmpy2.isqrt(n) + 1
    b = a**2 - n
    round = 0
    while not gmpy2.iroot(b, 2)[1]:
        round += 1
        print(f"Round {round}: a = {a}, b = {b}", end="\r")
        a += 1
        b = a**2 - n
    b = gmpy2.iroot(b, 2)[0]
    return (a + b, a - b)

However, I still couldn’t decompose ppp and qqq for a long time after I started running, maybe because ppp and ∣p−q∣|p-q|∣p−q∣ are actually not small.

#!/usr/local/bin/python3

import os
from flask import Flask, send_file, request, redirect, jsonify, render_template
import json
import string


def translate(command: str) -> str:
    emoji_table = json.load(open("emoji.json", "r", encoding="utf-8"))
    for key in emoji_table:
        if key in command:
            command = command.replace(key, emoji_table[key])
    return command.lower()


app = Flask(__name__)


@app.route("/")
def index():
    return render_template("index.html")


@app.route("/api")
def api():
    command = request.args.get("command")

    if len(set(command).intersection(set(string.printable.replace(" ", "")))) > 0:
        return jsonify({"command": command, "result": "Invalid command"})
    command = translate(command)
    result = os.popen(command + " 2>&1").read()
    return jsonify({"command": command, "result": result})


if __name__ == "__main__":
    app.run("0.0.0.0", 5000)

Quantum Nim Heist

This question is a Nim game. I checked it online and found that it has a sure-win method (a bit like the game I played when I was a kid, whoever calls 21 first wins). So I wrote the following exploit script:

from pwn import *
from typing import Tuple
import random
import time


def get_move(piles) -> Tuple[int, int]:
    nim_sum = 0
    for pile in piles:
        nim_sum ^= int(pile)

    if nim_sum == 0:
        # losing game, make a random move
        pile = random.randint(0, len(piles) - 1)
        count = random.randint(1, int(piles[pile]))
    else:
        # winning game, make a winning move
        for i, v in enumerate(piles):
            v = int(v)
            target = v ^ nim_sum
            if target < v:
                pile = i
                count = v - target
                break

    return (pile, count)

After running it several times, I found out why it always lost? Did I write it wrong?

After chanting ChatGPT several times, I found that I did not write it wrong. The problem was that I did not see the source code for the question… After looking at it, I found that the original question was destined to result in the first player (us) losing when the game was first generated. as follows:
